# 01 · 愿景与 Agent 分级

## 1. 一句话愿景

> 让广告主只需要给出**预算 + 目标 + 创意素材**，Adex Agent 自主完成跨平台投放、出价、预算分配、创意轮换和效果归因。

## 2. 目标用户与价值

| 用户 | 痛点 | Adex Agent 的承诺 |
|---|---|---|
| 中小品牌主 | 没人懂 Google/Meta/TikTok 后台，外包烧钱 | 不用学平台后台，只看一份周报 |
| 出海游戏 / 应用 | 多账号 + MMP（AppsFlyer/Adjust）数据散乱，调价 24h 滞后 | 跨平台统一看板 + 小时级自动调价 |
| Agency 投手 | 100 条 campaign 手工巡检 90% 是机械操作 | Agent 把巡检自动化，人只做策略和审批 |

## 3. 范围（In / Out）

### In Scope
- 跨平台账号接入（Google / Meta / TikTok / Amazon / LinkedIn / AppsFlyer / Adjust）
- 跨平台数据统一与归因（基础版，非 MTA）
- AI 创意生成（图/视频/文案）
- **Agent 化的投放执行**（本方案核心）
- 团队协作 / 审计 / 审批

### Out of Scope（明确不做）
- 多触点归因建模（MTA / MMM）— 留给专业 BI
- 自有 DSP / SSP / RTB
- Brand safety / 创意合规审查 — 接第三方
- 落地页 A/B 测试 — 留给 Optimizely / VWO
- CRM / 用户画像建设

## 4. Agent 自治等级（L0–L4）

借用自动驾驶分级思路，明确每一级的「人 vs Agent 责任边界」。

| 等级 | 名称 | 人的责任 | Agent 的责任 | 类比 |
|---|---|---|---|---|
| **L0** | 报表 | 全部 | 只读取数据 | 当前代码的 90% |
| **L1** | 建议 | 决策 + 执行 | 给出建议（不执行） | 当前 Advisor |
| **L2** | 单击审批 | 审批每一个动作 | 提议 + 执行（人点确认） | 当前 Advisor.apply（仅 pause/resume） |
| **L3** | 半自治 | 设定 guardrails + 异常审批 | 在 guardrails 内自主执行 | **本方案 12 个月内目标** |
| **L4** | 全自治 | 设定季度预算与目标 | 全部 | 24 个月外，需大量真实战绩 |

### L3 的具体含义（核心目标）

- 用户在 onboarding 时设置 guardrails：
  - 单日总预算上限
  - 单 campaign 调价幅度上限（如 ±30%/天）
  - 必须人工审批的动作类型（如「新建 campaign」「日预算 > $500」）
- Agent 每小时跑一次决策循环，在 guardrails 内自主执行：
  - 暂停 ROAS < 阈值的 campaign
  - 给 ROAS > 阈值的 campaign 加预算
  - CTR < 阈值时生成 + 推送新创意
  - 周期性 A/B 测试创意
- 触发 guardrail 的动作进入「待审批队列」，Slack/邮件通知。
- 每周自动生成「Agent 周报」：做了什么、为什么、效果如何、人工修正了什么。

## 5. 成功指标（North Star）

- **运营效率**：每位投手可管理 campaign 数 ≥ 现状的 5 倍。
- **响应速度**：从指标恶化到执行调整的中位时间 ≤ 1 小时（现状 ≥ 24h）。
- **执行准确率**：Agent 自主执行的动作中，被人工事后回滚的比例 ≤ 5%。
- **ROAS 提升**：在相同预算下，启用 Agent 的账号 30 天 ROAS 中位数提升 ≥ 15%。

## 6. 非目标 / 反模式

- ❌ **不做"全自动黑盒"**：每个动作必须可解释、可追溯、可回滚。
- ❌ **不假装比平台原生 AI 强**：Smart Bidding 等平台原生能力优先使用，Agent 只做平台之间的协调和平台不擅长的事（如跨平台预算重分配）。
- ❌ **不绕开平台政策**：所有动作都通过官方 API，不做爬虫、不做反作弊。
- ❌ **不替代投手判断**：Agent 是放大器，不是替代品；策略层（产品定位、品牌调性）始终是人的工作。
